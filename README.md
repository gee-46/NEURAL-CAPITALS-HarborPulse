# 📌 Overview

HarborPulse is an AI-driven harbor management platform designed to optimize dock scheduling, vessel coordination, unloading operations, and real-time harbor traffic management during peak landing hours.

The system transforms traditional harbor operations into a digitally coordinated ecosystem capable of handling high-volume maritime activity efficiently and sustainably.

By integrating smart scheduling algorithms, live operational monitoring, and scalable infrastructure design, HarborPulse aims to improve operational efficiency while minimizing delays, congestion, and spoilage in coastal logistics networks.

---

# 🚨 Problem Statement

During peak fishing and cargo landing periods, harbors often face:

* Severe dock congestion

* Inefficient unloading coordination
* Manual dock allocation processes
* Operational communication gaps

* Lack of centralized real-time visibility

Traditional systems are unable to dynamically adapt to fluctuating vessel traffic and operational constraints.

# 🧠 The "Pulse Score" Algorithm
Unlike static queues, HarborPulse uses a dynamic weighting formula:
**Score = (Wait Time × 0.5) + (Perishability × 10) - (Distance × 2)**

*   **Dynamic Perishability:** Prioritizes high-value cargo (Tuna/Prawns) to prevent economic loss.
*   **Distance Awareness:** Penalizes distant boats to ensure docks are never sitting empty.
*   **Capacity Tiebreaker:** Prioritizes smaller vessels to maximize harbor throughput.

# ⚙ Installation & Setup

## Clone Repository

```bash
git clone https://github.com/gee-46/NEURAL-CAPITALS-HarborPulse.git
cd NEURAL-CAPITALS-HarborPulse
```

---

# 📦 Frontend Setup

```bash
cd frontend
npm install
npm run dev
```

---

# 📦 Backend Setup

```bash
cd backend
npm install
npm start
```

---

# 🔐 Environment Variables

Create a `.env` file inside the backend directory.

```env
PORT=5000
MONGODB_URI=your_database_url
JWT_SECRET=your_secret_key
```
